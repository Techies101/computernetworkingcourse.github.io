IPv6

The ongoing expansion of the Internet and the need to provide IP addresses to accommodate it to support an increasing number of new users, computer networks, Internet-enabled devices, and new and improved collaboration and communication applications is accelerating the emergence of a new IP protocol.IPv6, with its robust architecture, was designed to meet these current and anticipated near-term needs.

Today, IP version 4 (IPv4) is widely used for the Internet, intranets, and private networks around the world. IPv6 improves on IPv4's functionality and structure in the following ways:

  • uAllows for more efficient routing by providing a simplified and enhanced packet header.
  • uImproves mobile phone and other mobile computing device support.
  • uIPsec is used to enforce increased, mandatory data security (which was originally designed for it).
  • uOffers more comprehensive quality-of-service (QoS) support.

IPv6 addresses are 128 bits long, as opposed to 32 bits long, and include a scope field that identifies the type of application that can use the address. IPv6 does not support broadcast addresses and instead uses multicast addresses to broadcast. In addition, IPv6 introduces a new address type known as anycast.
