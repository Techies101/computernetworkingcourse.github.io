Default Gateway

If a TCP/IP computer needs to communicate with a host on another network, it will usually do so via a device known as a router. A default gateway is a TCP/IP router that is specified on a host and connects the host's subnet to other networks.

This section describes how TCP/IP determines whether to send packets to its default gateway in order to reach another computer or device on the network.

When a host tries to communicate with another device via TCP/IP, it compares the defined subnet mask and the destination IP address to the subnet mask and its own IP address. The outcome of this comparison informs the computer whether the destination is a local or remote host.

If the outcome of this process indicates that the destination is a local host, the computer will send the packet on the local subnet. If the result of the comparison determines that the destination is a remote host, the packet is forwarded to the computer's default gateway as defined in its TCP/IP properties. It is then the router's responsibility to forward the packet to the correct subnet.
